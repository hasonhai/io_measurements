From this point we will only working on data of one host. We can select the host here. The tests performed on the 5 servers are the same. So the figures for one host is enough, otherwise it would be redundant. I intended to use the results on other servers for a reference of our results. From these data, I'm thinking we can develop some kind of factor that represents the interference on the cluster. Kind of "similarity coefficience". If the similarity (from the result) between the 5 servers is little, then we have too much interference or the figure from one server cannot be a representative. But if similarity is high, then we can conclude that we don't have much interefence that affects on our study.

The storage review suggest that our hard drives having peak speed at 115 MB/s for sequenctial read/write and 57 MB/s for random read/write with the test of 4K block size. However, we got triple the rate!?!?!? This transfer rate is like a modern Solid State Drive (SSD), not an hard drive with only 7200rpm.

http://www.storagereview.com/seagate_constellation2_and_constellation_es2_hard_drive_review

One explanation for this maybe because of the IO depth that they are using. When we run the same test with I/O depth at 1 for with native asynchonous I/O library of Linux, we also got around this value. Or maythe program that we use got a bug when reporting.
